Terraform Variables Deep Dive 🚀

Understand the WHY, HOW & FLOW like a real DevOps Engineer

📂 Why Different .tf Files?

main.tf

Contains actual resource logic (WHAT to create)

variables.tf

Defines inputs (WHAT values are needed)

terraform.tfvars

Stores actual values (WHAT we pass)

👉 Separation = Clean code + reusable infra + easy debugging 👉 This is exactly how production Terraform is written

❌ Why NOT Hardcoding?

resource "azurerm_resource_group" "rg" {
  name = "my-prod-rg"
}

Problems:

  • Not reusable ❌
  • Hard to change ❌
  • Breaks multi-environment setup ❌
✅ Correct Way:
name = var.rg_name

🔁 Terraform Variable Flow

1️⃣ Declare Variable
⬇️
2️⃣ Use Variable
⬇️
3️⃣ Assign Value

Step 1: Declare (variables.tf)

variable "rg_name" {
  type        = string
  description = "Resource Group Name"
}

Step 2: Use (main.tf)

resource "azurerm_resource_group" "rg" {
  name     = var.rg_name
  location = "West Europe"
}

Step 3: Assign Value (terraform.tfvars)

rg_name = "my-dev-rg"

⚡ Variable Priority (Important)

🥇 CLI

terraform apply -var="rg_name=prod"

🥈 tfvars

terraform.tfvars

🥉 ENV

TF_VAR_rg_name

🏁 Default

variables.tf

👉 Highest priority always overrides lower ones 👉 CLI will override everything

✅ Final Summary

  • Separate files = clean & scalable
  • Never hardcode values
  • Follow 3-step flow (Declare → Use → Assign)
  • Understand priority (CLI > tfvars > ENV > default)
