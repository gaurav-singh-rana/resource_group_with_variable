Terraform Variables – Complete Guide 🚀

Understand Variables using Real Flow (Declare → Use → Assign → Override)

📌 Basic Resource Block

resource "azurerm_resource_group" "resource_group" {
  name     = var.rg_name
  location = var.rg_location
}

Instead of hardcoding values, we use variables to make infrastructure dynamic and reusable.

📦 Variable = "Dabba" (Container)

Variable ek container hai jisme value store hoti hai. Pehle variable declare karna padta hai: 

variable "rg_name" {}
variable "rg_location" {}

🔁 Terraform Variable Flow

1️⃣ Declare Variable
⬇️
2️⃣ Use Variable
⬇️
3️⃣ Assign Value
⬇️
4️⃣ Override Value

Step 1: Declare Variable

variable "rg_name" {
  type = string
}

Step 2: Use Variable

name = var.rg_name
location = var.rg_location

Rule: Always use var. prefix

Step 3: Default Value

variable "rg_name" {
  default = "elephant"
}

If no value is passed → Terraform uses default

Step 4: terraform.tfvars

rg_name     = "ghoda"
rg_location = "eastus"

Terraform automatically reads this file

Step 5: CLI Override

terraform apply -var="rg_name=prod" -var="rg_location=westus"

CLI always overrides everything

⚡ Variable Priority (Important)

🥇 CLI (Highest)
🥈 terraform.tfvars
🥉 Default

👉 Highest priority always overrides lower values

❌ Why NOT Hardcoding?

  • Not reusable
  • Not scalable
  • Breaks multi-environment setup
✅ Why Variables?
  • Reusable code
  • Multi-environment support
  • Flexible & dynamic infra

📂 Production File Structure

main.tf            → Resource logic
variables.tf       → Variable declaration
terraform.tfvars   → Values

✅ Final Summary

  • Always declare variables first
  • Use var.variable_name to access
  • Assign values using tfvars or CLI
  • Understand priority (CLI > tfvars > default)
