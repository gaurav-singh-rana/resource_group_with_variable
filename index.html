Terraform Variables 🚀

Write Clean, Flexible & Scalable Infrastructure Code

📌 What are Terraform Variables?

Terraform variables are used to make your infrastructure code dynamic. Instead of hardcoding values, variables allow you to reuse the same code across different environments.

🔥 Why Variables Matter

```
♻️ Reusability

Write once and reuse across multiple environments.

⚡ Flexibility

Change inputs without modifying core infrastructure code.

🌍 Multi-Environment

Separate configurations for dev, staging, and production.

```

🧱 Types of Variables

  • String
  • Number
  • Boolean
  • List
  • Map

💻 Example

```
variables.tf
```


variable "resource_group_name" {
description = "Resource Group Name"
type        = string
}
```
terraform.tfvars
```


resource_group_name = "my-rg"
```
main.tf
```


resource "azurerm_resource_group" "rg" {
name     = var.resource_group_name
location = "Central India"
}

✅ Benefits

```
🚀 Faster Deployment

No need to rewrite code for every environment.

🧠 Maintainability

Clear separation of logic and configuration.

🔐 Security

Sensitive values can be handled safely.

📦 Clean Code

No hardcoded values makes code readable.

```

⚠️ Best Practices

  • Avoid hardcoding values
  • Use descriptive variable names
  • Use default values when needed
  • Keep environment configs in tfvars
